下面给你一套“新手能用起来、同时不踩坑”的 **AES-256-GCM** 全面讲解（概念 → 用法 → Go 代码 → 常见错误）。

---

## 1) AES-256-GCM 是什么

**AES-256-GCM = AES（对称加密算法）+ 256 位密钥 + GCM 模式（同时提供加密与完整性校验）**

* **AES**：对称加密（加密和解密用同一把密钥）。
* **256**：密钥长度 256 bit = 32 字节。
* **GCM**：一种 AEAD 模式（Authenticated Encryption with Associated Data）

  * 不仅“保密”（别人看不懂）
  * 还“防篡改”（密文被改过会解密失败）

对比一下：

* 只加密不认证（例如某些旧模式）可能被“修改密文导致明文变化”。
* GCM 会附带一个 **认证标签 tag**（一般 16 字节），解密时会验证 tag，验证不过就报错。

---

## 2) GCM 里最关键的三个东西：Key / Nonce / Tag

### Key（密钥）

* 32 字节（AES-256）
* 必须保密
* 一般来自配置/密钥管理系统（KMS），不要写死在代码里

### Nonce（也叫 IV）

* 每次加密都要一个 nonce，**必须唯一**（在同一个 key 下不能重复）
* 通常 12 字节（GCM 常见 nonce size = 12）
* 一般做法：**每次加密生成随机 nonce，然后把 nonce 和密文一起存/传**

> 重点：**nonce 复用是灾难级安全漏洞**（同一 key 下重复 nonce，会让攻击者推断明文甚至恢复密钥相关信息）。
> 你之前的测试 `EncryptUsesRandomNonce` 就是在确保这点。

### Tag（认证标签）

* GCM 的“防篡改凭证”
* 在 Go 里它通常跟密文一起返回（不是单独字段），长度默认 16 字节（128 bit tag）

---

## 3) AAD（Additional Authenticated Data）是什么

**AAD 不会被加密，但会被认证**：

* 你可以把“字段名、版本号、用户ID、协议头”等作为 AAD
* 加密时把 AAD 参与认证
* 解密时必须提供完全一样的 AAD，否则解密失败

你那份代码里 `fieldAAD(field)` 就是典型用法：把不同字段绑定到不同 AAD，防止“real_name 的密文拿去当 student_id 解”。

---

## 4) AES-256-GCM 的标准使用流程

### 加密流程

1. 准备 32 字节 key
2. 初始化 AES block
3. 用 block 初始化 GCM
4. 生成随机 nonce（长度 = gcm.NonceSize()）
5. 调用 `Seal` 得到 ciphertext（内部包含 tag）
6. 输出：通常是 `nonce || ciphertext`（拼起来），再做 base64 便于存储传输

### 解密流程

1. 解析出 nonce 和 ciphertext
2. 调用 `Open` 解密，并验证 tag + AAD
3. 如果 nonce/AAD/密文/tag 任一不对 → `Open` 返回错误

## 5) Go 语言最小可用示例（加密/解密）

```go
package main

import (
	"crypto/aes"
	"crypto/cipher"
	"crypto/rand"
	"encoding/base64"
	"errors"
	"fmt"
	"io"
)

func EncryptAESGCM(key []byte, plaintext []byte, aad []byte) (string, error) {
	if len(key) != 32 {
		return "", fmt.Errorf("key must be 32 bytes (AES-256), got %d", len(key))
	}

	block, err := aes.NewCipher(key)
	if err != nil {
		return "", err
	}

	gcm, err := cipher.NewGCM(block)
	if err != nil {
		return "", err
	}

	nonce := make([]byte, gcm.NonceSize())
	if _, err := io.ReadFull(rand.Reader, nonce); err != nil {
		return "", err
	}

	// Seal 返回: ciphertext || tag（tag 在末尾，由 GCM 管理）
	ciphertext := gcm.Seal(nil, nonce, plaintext, aad)

	// 输出 nonce + ciphertext，方便解密时取回 nonce
	out := append(nonce, ciphertext...)
	return base64.StdEncoding.EncodeToString(out), nil
}

func DecryptAESGCM(key []byte, b64 string, aad []byte) ([]byte, error) {
	if len(key) != 32 {
		return nil, fmt.Errorf("key must be 32 bytes (AES-256), got %d", len(key))
	}

	raw, err := base64.StdEncoding.DecodeString(b64)
	if err != nil {
		return nil, err
	}

	block, err := aes.NewCipher(key)
	if err != nil {
		return nil, err
	}

	gcm, err := cipher.NewGCM(block)
	if err != nil {
		return nil, err
	}

	nonceSize := gcm.NonceSize()
	if len(raw) <= nonceSize {
		return nil, errors.New("ciphertext too short")
	}

	nonce := raw[:nonceSize]
	ciphertext := raw[nonceSize:]

	// Open 会同时校验 tag + aad，任何不匹配都会报错
	plaintext, err := gcm.Open(nil, nonce, ciphertext, aad)
	if err != nil {
		return nil, err
	}

	return plaintext, nil
}

func main() {
	key := make([]byte, 32)
	if _, err := rand.Read(key); err != nil {
		panic(err)
	}

	aad := []byte("field:real_name:v1")
	enc, err := EncryptAESGCM(key, []byte("张三"), aad)
	if err != nil {
		panic(err)
	}
	fmt.Println("enc:", enc)

	dec, err := DecryptAESGCM(key, enc, aad)
	if err != nil {
		panic(err)
	}
	fmt.Println("dec:", string(dec))
}
```

你能看到核心就两句：

* 加密：`gcm.Seal(nil, nonce, plaintext, aad)`
* 解密：`gcm.Open(nil, nonce, ciphertext, aad)`

## 6) 常见坑（新手最容易踩）

### 坑 1：nonce 复用（最危险）

**同一个 key 下 nonce 绝对不能重复。**
安全做法：每次随机生成 nonce，然后把 nonce 和密文一起保存。

### 坑 2：AAD 不一致导致解密失败

加密用的 AAD 和解密必须完全一致（字节级一致）。
你代码里用固定字符串 `verify:student_id:v1` 这种就不容易错。

### 坑 3：把 base64/hex 搞混

* base64：更短，适合存储/传输二进制
* hex：更长（2倍），但更直观
  你那份代码是：**密文用 base64**，哈希用 hex（因为哈希常以 hex 展示）。

### 坑 4：以为“加密后还能搜索/去重”

**加密是可逆但不可搜索的（同明文每次密文不同）**
如果要去重/索引，应该像你们做的那样用 **HMAC 哈希**（稳定输出）。

### 坑 5：密钥管理

* 不要把 key 写死在代码仓库
* 不要打印 key 或密文原始字节到日志
* 生产环境建议走配置中心/KMS，并支持轮换（key rotation）

* 学号索引用 **HMAC-SHA256**（不可逆但可匹配）
